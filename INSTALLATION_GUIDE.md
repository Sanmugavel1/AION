## Step 1 — Enable virtualization in BIOS (manual, requires reboot)

This is the hard blocker. It cannot be done from Windows or by me — it's a firmware setting.

1. Restart the laptop.
2. As soon as the HP logo appears, repeatedly press **F10** to enter BIOS Setup (HP's standard key; if it doesn't catch, try **Esc** then **F10**).
3. Go to **Advanced → System Options** (menu name varies slightly by BIOS revision).
4. Find **Virtualization Technology (VTx)** / **Intel Virtualization Technology** and set it to **Enabled**.
5. If present, also enable **VT-d**.
6. Press **F10** to save and exit, confirm reboot.

## Step 2 — Verify virtualization is now on

Once back in Windows, open PowerShell and run:

```powershell
(Get-CimInstance Win32_Processor).VirtualizationFirmwareEnabled
```

Must return `True` before continuing.

## Step 3 — Enable WSL2 and Virtual Machine Platform (elevated PowerShell)

Open PowerShell **as Administrator** (Start menu → search "PowerShell" → Run as administrator), then:

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

Restart Windows when prompted, then set WSL2 as the default version and install a kernel update:

```powershell
wsl --set-default-version 2
wsl --update
```

(Equivalently, `wsl --install` does most of this in one shot on a machine that has never had WSL — but since virtualization was off, doing it in two explicit passes avoids a half-finished state.)

## Step 4 — Install Docker Desktop

1. Download the installer: https://www.docker.com/products/docker-desktop/
2. Run `Docker Desktop Installer.exe`.
3. Keep **"Use WSL 2 instead of Hyper-V"** checked (default on Home edition).
4. Finish install, reboot if prompted, then launch Docker Desktop and let it finish its first-run setup.
5. Accept the license agreement when it appears.

## Step 5 — Verify Docker

```powershell
docker --version
docker run hello-world
```

Both must succeed before I resume the AION startup automation.
